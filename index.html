log_roller by Jacob Vorreuter

A distributed Erlang publishing and logging system


Connected Erlang nodes can act as log publishers, consumers or both. Publishers are responsible for broadcasting log messages that pass through their error_logger to any listening nodes. Consumers are responsible for logging messages they receive to disk via the disk_log module. Starting the log_roller application registers the log_roller_h report handler with the error_logger event manager. The log_roller_h gen_event server receives error_logger messages and passes them to connected consumer nodes. Consumer nodes are identified by the presence of a running log_roller_disk_logger server, which is started when the log_roller_server application starts. <%--Both publisher and consumer nodes perform an auto-discovery routine that involves pinging connected nodes to establish publisher/consumer relationships.--%>

About

log_roller is based on a publisher/subscriber model. It integrates seamlessly with existing systems without the need to change code. Nodes designated as publishers broadcast log messages sent to the error_logger to nodes designated as subscribers. The subscriber writes the log message to a rotating disk log similar to the SASL error_logger_mf_maxbytes and error_logger_mf_maxfiles functionality. The subscriber also contains a log browser and a pre-configured webtool interface for monitoring logging in a browser.

Publishers and subscribers perform auto-discovery upon booting. During startup, nodes of either type (publisher or subscriber) will attempt to connect. Make sure to have matching ~/.erlang.cookie files hosted on the servers running log_roller. The nodes will not be able to see each other otherwise.
